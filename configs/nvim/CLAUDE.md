# Neovim Config

## Structure

- `init.lua` -- entry point (leader key + requires)
- `lsp/` -- Neovim 0.12 native LSP configs (each returns `{ cmd, filetypes, root_markers }`)
- `lua/config/` -- options, keymaps, autocmds, pack (vim.pack), statusline, LSP activation
- `lua/plugins/` -- imperative plugin setup files (loaded by pack.lua in dependency order)

## Load Order

`mapleader` before requires. `config.pack` loads all plugins and then requires each `plugins/*.lua` file. `config.lsp` after `config.pack` (needs blink.cmp).

## Rules

- Plugin-specific keymaps go in the plugin's setup file in `lua/plugins/`
- General keymaps go in `lua/config/keymaps.lua`
- New plugins: add URL to `vim.pack.add()` in `lua/config/pack.lua`, add setup file in `lua/plugins/`, add `require()` in pack.lua
- New LSP servers: add `lsp/<name>.lua` (return config table) + add name to `vim.lsp.enable()` in `lua/config/lsp.lua`
- All autocmds use the `user_config` augroup
- `nvim-pack-lock.json` is committed (generated by vim.pack)
- Use `buf` (not deprecated `buffer`) in `vim.keymap.set` opts

## Verify

No build step. Open `nvim` and check `:checkhealth`.

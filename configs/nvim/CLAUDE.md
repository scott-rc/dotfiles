# Neovim Config

## Structure

- `init.lua` -- entry point (leader key + requires)
- `lsp/` -- Neovim 0.11 native LSP configs (each returns `{ cmd, filetypes, root_markers }`)
- `lua/config/` -- options, keymaps, autocmds, lazy bootstrap, LSP activation
- `lua/plugins/` -- lazy.nvim auto-discovers all files here

## Load Order

`mapleader` before requires. `config.lsp` after `config.lazy` (needs blink.cmp).

## Rules

- Plugin-specific keymaps go in plugin spec `keys` field
- General keymaps go in `lua/config/keymaps.lua`
- New plugins: add file in `lua/plugins/`, return a lazy.nvim spec table
- New LSP servers: add `lsp/<name>.lua` (return config table) + add name to `vim.lsp.enable()` in `lua/config/lsp.lua`
- Neo-tree autocmds use `neotree_config` augroup; everything else uses `user_config` augroup
- `lazy-lock.json` is gitignored (generated by lazy.nvim)

## Verify

No build step. Open `nvim` and check `:Lazy` and `:checkhealth`.
